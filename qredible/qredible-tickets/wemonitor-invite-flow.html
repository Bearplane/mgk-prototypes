We Monitor Invites — Bulk Invite + Feature-Scoped Link (MyCOA Wizard + Paywall)

The Invites tab in We Monitor gains an Active/Pending source filter, multi-select with a bulk Invite action, and a feature-scoped invite dialog. When the invite is scoped to MyCOA, the generated link routes the recipient into the MyCOA Wizard, and the downstream paywall filters to MyCOA-eligible plans only.

ℹ INFO — NOT IN BUILD

How to read this mockup: Yellow dashed blocks are annotations for Joseph / design review — never rendered in-app. Everything outside these blocks is ship-ready implementation copy. [NEEDS COPY] flags genuine gaps.

  • Feature selector is single-select, MyCOA first. Future features (Q-ScanAI self-serve, Policy Portal, etc.) appear under a "Coming soon" group but are disabled until a Q-Config plan exposes them. MyCOA is the only feature live at launch.
  • Invite source = the Active + Pending company lists. Invites tab doesn't create new company records; it selects from what We Monitor already knows about. This matches the ticket language "populate companies".
  • Link scope is persisted on the invite token (invites.feature_scope = 'mycoa') — the paywall and wizard branch off this scope server-side, so a link leaked outside its intended feature still funnels correctly.
  • Email template toggle exposes MyCOA-specific copy blocks (subject line, CTA label, body paragraph). Default template is feature-agnostic; flipping the MyCOA toggle swaps to MyCOA copy.
  • Paywall behavior in this mockup is illustrative. Ticket 2 (Q-Config) is the actual source of plan→feature mapping. See the annotation callout in Example 5.

Example 5 — Annotation · Recipient flow when MyCOA is scoped
Callout: recipient flow when the invite is scoped to MyCOA
Callout When MyCOA is the scoped feature on an invite:
  1. The recipient's sign-up completes and they are routed directly into the MyCOA Wizard (skipping the generic post-signup dashboard).
  2. The Wizard collects the merchant's first COA upload, display preferences, and domain/CNAME intent.
  3. On Wizard completion they hit the paywall, which — because invites.feature_scope = 'mycoa' — shows only MyCOA-eligible plans. Plans that do not expose the MyCOA feature flag (Ticket 2) are hidden.
  4. Plan selection and payment finalize the MyCOA subscription; the published MyCOA page goes live.
