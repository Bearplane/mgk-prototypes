QP-3794 · NEW Mirror Product Compliance Products tab into MyCOA, scoped to the merchant’s own products.
ℹ DESIGN NOTES — NOT IN BUILD
  • Tab order: Display | Products (new — this ticket) | COAs. Products is the new tab added here; Display and COAs are unchanged.
  • Shared component: The Products tab body is the same component as Product Compliance → Products tab in Q-Monitor, reused with viewerRole: "merchant".
  • Company column: Hidden/pinned in merchant view — the merchant only sees their own products, so the Company column is suppressed entirely.
  • Q-Config gate: Gated behind new mycoa_products_tab Q-Config Source; default value: disabled. Enable per-account in Q-Config.
  • Default landing tab: Any change to the default landing tab (opening to Products instead of Display) is tracked separately in QP-3745 and is NOT part of this ticket.
  • Processor-only actions NOT shown: “+ Add to case”, Bulk Edit, Archive, Delete.
  • Merchant-visible actions: Add Product, Import Products, Scan for Products, Report an Issue.

Section 3 · Action Bar
Action bar visibility by viewerRole
Merchant-visible actions (included in this ticket):
  • Add Product — opens the add/create product flow
  • Import Products — opens CSV/XLSX import wizard
  • Scan for Products — triggers a Q-ScanAI discovery scan for this merchant
  • Report an Issue — opens the merchant issue-report form

Explicitly omitted (processor-only, NOT shown to merchant):
  • + Add to case — processor workflow; merchant has no case concept
  • Bulk Edit — processor-only bulk management action
  • Archive — processor-only lifecycle action
  • Delete — processor-only destructive action

The action bar is the primary surface where viewerRole drives button visibility. All omissions are additive defaults — the processor view retains all buttons; merchant view is the restricted subset.
Section 4 · Q-Config Gate
Feature flag & default landing tab
Q-Config Source: mycoa_products_tab Default: Disabled

The Products tab is gated behind the mycoa_products_tab Q-Config Source. Default value is disabled. The tab is not surfaced to any merchant until the flag is enabled per-account in Q-Config. Enable per-account (or globally) in Q-Config to surface the Products tab in MyCOA.

Out of scope — QP-3745: Default landing tab change (opening MyCOA to the Products tab instead of Display by default) is tracked separately in QP-3745 and is not part of this ticket. This ticket only adds the Products tab; it does not change which tab opens by default.
