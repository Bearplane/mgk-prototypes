Problem

BPO team has no structured way to categorize invalid COAs. No distinction between "not a COA at all" vs "insufficient data" vs "suspicious." No distinction between system-detected flags and human-confirmed issues.

Issue Categories
Not a COA Out-of-Scope Insufficient Data Suspicious COA Lab Error Duplicate Unreadable
Business Rules
BR-17 Issue Category values as listed above.
BR-2System-flagged (auto): "System Flagged" badge + Confirm/Dismiss inline. Human-confirmed: "Validation Issue" badge + category tooltip.
BR-3Confirmed Issue Category → new status "Validation Issue", separate from Violative/Expired/Missing/Compliant.
BR-4Selectable from three-dot "Flag Issue" and from COA Details modal.
BR-5Tooltip on Validation Issue badge shows category reason, reviewer, timestamp.
BR-6"Validation Issue" added to Compliance Issue filter dropdown.
Acceptance Criteria
☐Issue Category exists with all 7 values
☐"Validation Issue" COA status distinct from all others
☐BPO can flag from three-dot menu and COA Details modal
☐Badge tooltip shows category + reviewer + timestamp
☐System-flagged shows "System Flagged" badge with Confirm/Dismiss
☐"Validation Issue" filterable in Compliance Issue filter
☐Validation Issue COAs excluded from compliant counts
