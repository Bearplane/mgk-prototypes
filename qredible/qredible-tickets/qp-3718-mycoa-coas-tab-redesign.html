MyCOA COAs Tab — Redesign

The COAs tab is the merchant's daily surface for triaging incoming lab results. Today it hides the things they actually decide on — batch #, collection date, analyte pass/fail, disposition — behind rows and separate flyouts, and forces a two-step publish (MyCOA first, then Q-Report). This mockup surfaces those fields as first-class columns and collapses the publish flow into a single unified action with a preview of what each destination receives.

MyCOA · COAs tab List view cleanup Unified publish (MyCOA + Q-Report)

Problem

  • Batch # and collection date are buried — require opening each row
  • Analyte pass/fail surfaced only via a non-glanceable status dot
  • Disposition (Release / Hold / Quarantine / Reject) not shown in list
  • Publish is a two-step dance: Publish to MyCOA → then separately Push to Q-Report
  • Merchants forget to push to Q-Report → inconsistent public record

Solution

  • Promote Batch #, Collection Date, Analytes, Disposition to first-class columns
  • Analytes column shows compact pass/fail/pending bar + counts
  • Disposition rendered as a color-coded pill, sortable & filterable
  • Single Publish action opens a unified dialog with MyCOA + Q-Report side-by-side
  • Per-row pub-state dots make it obvious where each COA is live
