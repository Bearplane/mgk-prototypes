Resolved Product Decisions
✓
Resolution Types: custom or controlled?
Phase 1 = system defaults. Phase 2 = processor can ADD custom types. Automation hooks to violation type, not resolution type.
✓
SLA defaults
Manual per case at creation. Phase 2 may add processor-level defaults per priority.
✓
Dispute as merchant right
Yes — DISPUTE always available on COMPLIANCE_FINDING, COA_VIOLATIVE, LICENSE_ISSUE.
✓
Force Close permissions
Admin-only. Required note. Items archived as FORCE_CLOSED with audit trail.
✓
Extension request behavior
No auto-pause. Processor approves/denies. Approved = new SLA date. Both actions fire notification.
✓
Case number format
CR-YYYY-XXXX — auto-generated, tenant-scoped sequence per processor.
