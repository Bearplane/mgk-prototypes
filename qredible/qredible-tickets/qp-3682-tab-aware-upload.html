Tab-Aware Company Upload — Add Company + Bulk Upload Modals

WeMonitor surface: Add Company and Bulk Upload modals expose an Active / Pending destination-tab picker. Supports the Pending Q review workflow (v2.2, deploy 2026-04-23).

ℹ INFO — NOT IN BUILD

How to read this mockup: Yellow dashed blocks like this one are annotations for Joseph / design review — they are never rendered in the actual app. Everything outside yellow blocks is ship-ready implementation copy. [NEEDS COPY: …] flags mark genuine gaps that need a decision before implementation.

  • Key design decision: Destination tab is a segmented control (2 segments: Active / Pending), not a dropdown. Keeps it compact and instantly scannable. Archived is intentionally absent — the API rejects Archived as an initial destination.
  • Default = Pending in both modals, aligning with the Pending Q review workflow.
  • Bulk upload: one destination for the entire batch in v1. Per-row override is a future consideration per ticket.
  • Post-submit redirect: confirmation message names the destination tab and links to it directly.

ℹ INFO — NOT IN BUILD — Acceptance Criteria (from ticket)
QP-3682 — Test Panel
Add Company modal shows destination tab picker with Active and Pending options. Archived is not offered.
Bulk Upload modal shows same picker, with copy clarifying it applies to the whole batch.
Default selection is Pending in both modals.
Add Company → Active selected → company appears on Active tab.
Add Company → Pending selected → company appears on Pending tab.
Bulk Upload → Pending → every row lands in Pending tab.
Bulk Upload → Active → every row lands in Active tab.
API rejects destination = Archived on either flow.
Confirmation message names the destination tab and links to it.
Newly-uploaded Pending companies appear in the Pending Q workflow surface.
Destination field is visible without scrolling on a 1366×768 viewport.
ℹ INFO — NOT IN BUILD — Future Considerations (v1 deferred)
  • Per-row tab override in bulk upload — possible future CSV column (destination_tab). Not in v1.
  • Archived as a destination — excluded by design. Revisit only with concrete compliance justification.
  • Session-default memory — remember operator's last-used destination tab. Deferred.
  • Mixed-destination bulk upload — some Active, some Pending in one batch. Not scoped here; may surface in Bulk Team Member Assignment flow.
