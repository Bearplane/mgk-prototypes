MyCOA Display: Sticky Save Bar

Replaces the top-right "Save" button with a viewport-anchored save bar that slides up when the form is dirty. Per-section dots flag exactly which sections have unsaved changes.

Interaction guide
  • Clean state (page load): Save bar is hidden, no red dots on section headers.
  • Edit any field: Save bar slides up from bottom; a dot appears next to that section's heading.
  • "Save changes": 1s loading state → bar slides away → dots clear → "Saved ✓" toast (top-right, fades after 2 s).
  • "Discard": Confirmation prompt → all changes reset to clean state.
  • "Preview": Opens COA preview without requiring a save first — added specifically to reduce the current preview round-trip friction.
  • ⌘S / Ctrl+S: Keyboard shortcut triggers Save whenever the bar is visible.
Delta vs. current UI: The existing top-right "Save" button is removed. Everything below is the real Display tab layout; the save bar and dots are the only additions.
