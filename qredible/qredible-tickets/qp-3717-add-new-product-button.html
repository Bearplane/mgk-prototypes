Scope

In scope · QP-3717

  • Add "+ Add New Product" button to Product Assignment toolbar (right side, primary style)
  • Click opens the existing Product Builder modal — same instance used elsewhere
  • Newly-created product lands in the current tab with Just added badge and Unassigned state
  • Unassign action remains; both actions sit side-by-side

Out of scope

  • URL ingestion / "Add from product URL" flow → QP-3730
  • Any new fields or steps inside Product Builder
  • Bulk add / CSV import (already tracked under import-workflow)
  • Auto-assignment after create (product lands Unassigned; use existing assign flow)
↗
URL ingestion (QP-3730): a separate ticket adds a secondary entry — "Add from URL" — that scrapes a product page and pre-fills Product Builder. That surface belongs in QP-3730 and should not be built inside QP-3717.
