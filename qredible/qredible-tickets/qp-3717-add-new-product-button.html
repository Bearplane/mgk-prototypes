Add New Product — Product Assignment column

On the MyCOA COAs tab, the Product Assignment column shows either a product name or Unassigned. Clicking it today opens a modal with Reassign / Unassign buttons and a dropdown of existing products. There is no way to create a new product for a COA without leaving this flow. QP-3717 adds an Add New Product option that opens the existing Product Builder modal and auto-assigns the new product to this COA. QP-3730 (Import from URL) shares the same entry point.

Scope

What's in / out of this ticket

Out of scope

  • Product Builder itself — reused verbatim, no field changes
  • URL-ingestion logic — QP-3730
  • Reassign / Unassign behaviour — unchanged
  • Unified Publish toggle — QP-3718
