Bulk Team Member Assignment — Status Filters + Multi-Field Search

Extends the company-selection step of the existing Bulk Team Member Assignment stepper modal with status/team-member filters and legal name / DBA / website / MID search.

ℹ INFO — NOT IN BUILD

How to read this mockup: Yellow dashed blocks are annotations — never rendered in the app. Everything outside yellow blocks is ship-ready copy. [NEEDS COPY: …] flags mark gaps needing a decision before build.

  • Scope: Only Step 1 (company selection) of the existing 3-step Bulk Team Member Assignment wizard changes. Steps 2 and 3 are unchanged.
  • Filters are additive / composable: Status filter + team-member filter + search all narrow the list further when combined.
  • Selections persist across filter changes. If a selected company falls outside the current filter view, it stays selected — the counter surfaces "X selected, Y visible" to prevent silent deselection.
  • Clear filters resets all filters and search text but does NOT clear the selection.
  • Search is multi-field: fans out across pl_company.legal_name, pl_company_dba.dba_name, pl_company.website, pl_company_dba.merchant_identifier. Results deduped at pl_company_id level.
  • Default filter: Status = Active + Pending (Archived excluded by default).

ℹ INFO — NOT IN BUILD — Acceptance Criteria (from ticket)
QP-3683 — Test Panel
Status filter renders with Active, Pending, Archived options; defaults to Active + Pending.
Current team members filter renders as multi-select user picker with an Unassigned option.
Search input visible above the company list; placeholder names all four searchable fields.
Searching a substring of legal name returns matching companies.
Searching a substring of a DBA returns matching companies.
Searching a substring of a website URL returns matching companies.
Searching a full or partial MID returns matching companies.
A search matching a company via multiple fields returns the company once (deduped at pl_company_id).
Filters and search compose — applying status filter on top of search narrows results further.
Selecting Unassigned in team-member filter returns only companies with zero current assignees.
Existing selections persist across filter changes; counter shows "X selected, Y visible" when they diverge.
Clear filters resets all filters and search but does NOT clear the selection.
Downstream assignment step (Step 2 + Step 3) still works end-to-end with new filters in place.
Result count updates live as filters / search are applied.
ℹ INFO — NOT IN BUILD — Future Considerations (v1 deferred)
  • Saving filter presets per operator (e.g. "my Pending bucket"). Not in v1.
  • Server-side vs client-side search — depends on list size at runtime. May need server-side once list exceeds practical client threshold. Confirm with backend during implementation.
  • MID match context inline — whether to show which DBA the matched MID belongs to in the result row. Deferred.
  • Bulk-deselect within a filter view — "remove all currently visible" from selection. Not in v1.
  • Additional searchable fields (phone, address, license number). Defer until requested.
