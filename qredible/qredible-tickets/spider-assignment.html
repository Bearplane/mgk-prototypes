② Spiders Modal — List View.
Default Spider section: Read-only. Shows qredible_spider (type: template). Displays "All unassigned companies" — no edit or delete controls.
Custom Spiders section: Lists non-default spiders. Each row shows name, type badge, assigned company count, and whether location selection is required. Pencil icon opens the edit panel (Section ③). Trash icon deletes with confirmation.
"+ Add Spider" button opens the edit panel in create mode. Search bar filters both sections.

③ Spider Edit Panel. Clicking the pencil icon on a custom spider expands the modal to a split-panel layout.
Left panel: Lists all custom spiders for quick switching. "+ Add Spider" at bottom.
Right panel: Form fields — Spider Name and Type are read-only (sourced from Zyte). Zyte Spider ID is also read-only.
Requires Location Selection toggle: ON = when any assigned company triggers a scan, the system shows a location-picker modal first. OFF = scan proceeds immediately with the custom spider.
Assigned Companies: Searchable multi-select. Checked items = assigned to this spider. Unchecked items = use default spider. The 5 Black Bear locations are all assigned and location selection is ON.

④ Scan Trigger Flow. When "Scan for Products" is clicked for any company with a custom spider assigned:
• The system checks the spider assignment table for the company.
• If Requires Location Selection = ON, the location-picker modal fires before the scan (shown above).
• The user picks a store → scan fires using blackbeardispensary_spider for that location.
Blue info banner at the top of the Product Compliance tab informs the user that a custom spider is in use — this prevents confusion about why a modal appears.
Same behavior applies in We Monitor → Run Product Scan column. The Findings tab's "Custom Product Scan" button becomes redundant and can be deprecated.
⑤ Scan Trigger Behavior — Decision Logic
Company has custom spider? Requires Location Selection? Behavior when scan triggered Spider used
❌ No — Scan proceeds immediately. No modal. qredible_spider (default)
✅ Yes ❌ No Scan proceeds immediately with custom spider. No modal. Custom spider (e.g. acme_spider)
✅ Yes ✅ Yes Location-picker modal appears first. User selects location → scan fires. blackbeardispensary_spider
⑤ Decision logic applies universally across all scan entry points: Product Compliance tab → "Scan for Products", and We Monitor tab → "Run Product Scan". The Findings tab's "Custom Product Scan" button may be deprecated once this is live, as it becomes redundant for Black Bear.
