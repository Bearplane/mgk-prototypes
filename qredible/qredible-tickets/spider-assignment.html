③ Spider Edit Panel. Clicking the pencil icon on a custom spider expands the modal to a split-panel layout.
Left panel: Lists all custom spiders for quick switching. "+ Add Spider" at bottom.
Right panel: Form fields — Spider Name and Type are read-only (sourced from spider configuration). Spider ID is also read-only.
Spider Enum Values → Company Mapping: Replaces the old "Assigned Companies" checkbox list and the "Requires Location Selection" toggle. Each enum value (a string the spider accepts to target a specific location) maps to one or more companies. When a mapped company triggers a scan, the system automatically resolves the correct enum value and fires the scan immediately — no modal, no user decision needed. The "All" row is the default fallback: it scans all locations and cannot be deleted. "+ Add Enum Value" allows adding new location mappings as Black Bear expands.

④ One-Click Scan Flow. When "Scan for Products" is clicked for any company with a custom spider and an enum mapping:
• The system resolves the enum value automatically from the company → spider mapping table.
• No modal appears. The scan fires immediately with the correct spider and enum value.
• A green indicator next to the scan button shows the mapped spider and enum value for transparency.
• A success toast confirms the scan was initiated with the correct configuration.
The Findings tab's "Custom Product Scan" button (with its location dropdown) remains as the fallback for unmapped companies or for ad-hoc "All" scans — it is not deprecated, but it's no longer the primary path for mapped companies.
⑤ Scan Trigger Behavior — Decision Logic
Company has custom spider? Has enum mapping? Behavior when scan triggered Spider used
❌ No — Scan proceeds immediately. No modal. qredible_spider (default)
✅ Yes ✅ Mapped Scan proceeds immediately using mapped enum value. No modal. One click. Custom spider + mapped enum (e.g. blackbeardispensary_spider + Bridgeville)
✅ Yes ❌ Not mapped Scan proceeds using custom spider with "All" enum (default). No modal. Custom spider with All
⑤ Decision logic applies universally across all scan entry points: Product Compliance tab → "Scan for Products", and We Monitor tab → "Run Product Scan".
The Findings tab's "Custom Product Scan" button (with its location dropdown) becomes the fallback path for: (1) companies with custom spiders that have no enum mapping, or (2) admins who need to trigger an ad-hoc "All"-location scan outside the normal mapped flow. It is not deprecated — it serves a distinct purpose as the manual override and unmapped-company fallback.
