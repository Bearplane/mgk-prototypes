Q-Config Subscription Builder — Feature Selector on Plan Details (MyCOA first)

Adds a Features control to the Q-Config subscription plan Details screen. Operators tag each plan with one or more features (MyCOA first). The saved features[] array is consumed downstream by the We Monitor invite paywall (Ticket 1) to filter plans by feature scope.

ℹ INFO — NOT IN BUILD

How to read this mockup: Yellow dashed blocks are annotations for Joseph / design review — never rendered in-app. Fields marked Placeholder are pre-existing and out of scope for this ticket; only the Features control (green NEW badge) is net-new.

  • Features is a multi-select (plans may expose more than one feature), not a single radio. A plan with features = ["mycoa","qscanai"] appears at paywall for either scope.
  • MyCOA is the first and only enabled option at launch. Other features appear in the list but are disabled ("Coming soon") — they surface as the roadmap fills in.
  • Persistence: subscription_plans.features — text[] or jsonb. Key/slug per feature: mycoa, qscanai, policy_portal, etc. Stable slug, not the display name.
  • Saved state (Example 2) renders the feature tags on the plan card and in the plan list — so operators can scan which plans expose what without re-opening Details.
  • Integration with Ticket 1 (Example 3): callout explains the flag flow from Q-Config → invite token → paywall filter.

Example 3 — Annotation · Feature flag is consumed by the We Monitor invite paywall (Ticket 1)
How this field gets consumed downstream
Callout The subscription_plans.features array set here is the single source of truth for paywall filtering. When an invite is sent with a feature scope (Ticket 1), the paywall query is:
SELECT * FROM subscription_plans
WHERE status = 'published'
  AND 'mycoa' = ANY(features)
So a plan only shows at a MyCOA-scoped paywall if "mycoa" appears in its features array. Add/remove the MyCOA tag here and the paywall list updates on the next invite click — no code deploy.
① Q-Config (this ticket)
Operator tags MyCOA Starter with the mycoa feature and saves. Row is written to subscription_plans.features = ["mycoa"].
→
② We Monitor Invite (Ticket 1)
Operator sends an invite scoped to MyCOA. Token carries feature_scope = 'mycoa'. On paywall, only plans containing "mycoa" in features[] are shown — MyCOA Starter and MyCOA Growth qualify; Q-Audit Enterprise does not.
Rollout note: Ticket 2 ships before Ticket 1. Plans must be tagged before the first MyCOA-scoped invite goes out — otherwise the paywall returns an empty list and the merchant is stuck. A pre-flight check in Ticket 1's invite dialog (disabled "Send" button with a warning) will guard this; tracked as a follow-up task.
