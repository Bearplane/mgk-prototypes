Audit log rows for compliance report jobs are immutable. Each run produces a new compliance_report_job record linked to the merchant, policy version, and output report artifact.
System Prompt
Active policy JSON (hemp-cbd-base-v3 v12) is passed as the system prompt. Defines evaluation rules, section weights, scoring thresholds, and report configuration.
User Content
Merchant checklist evidence (34 items), product catalog (847 products with COA status and TypeDisposition), MRI results, and tier requirements passed as user message content.
web_search Tool
Live lookups: state cultivation license status, FDA warning letter search, OFAC/Vital4 MRI verification, real-time regulatory news for hemp/CBD. Results injected into evaluation context.
Policy-driven output shape: The report_configuration.sections[] array in the active policy JSON determines which sections appear in the rendered report, their order, and their scoring weights. Changing the policy changes the report shape — no code deploys required.
