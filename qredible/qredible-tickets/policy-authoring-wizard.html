Policy Authoring Wizard — Guided Component Builder

Step-by-step form that builds policy components without requiring JSON knowledge. Resulting JSON is appended to the policy's checklist[] or mri_configuration[] array. Walks through a complete License Verification component as the worked example.

① Context
Two modes in the Policy Editor
Expert JSON Mode EXISTING
Staff hand-edit the raw policy JSON document directly. Full flexibility — any valid policy shape is accepted. Requires understanding of the policy schema. Used by platform engineers and advanced compliance architects.
Accessed via: policy-editor.html → JSON tab
Guided Mode (this wizard) NEW · QP-3703
A step-by-step form wizard that builds the same JSON without requiring JSON knowledge. Compliance staff answer questions about what they need; the wizard assembles the component. After completing all steps, the resulting JSON component is appended to checklist[] or mri_configuration[].
Accessed via: Policy Editor → "Add Component" → "Use wizard"
Key principle: Both modes produce identical output. Editing a component in the wizard and editing it in JSON mode are equivalent operations. The wizard is a UX affordance — not a separate data model.

④ After Save — Two-Layer Model
Narrative view vs. machine-readable JSON
✅
Component saved successfully
ABC License Verification has been appended to checklist[] in policy hemp-cbd-base-v3. Next scan will include this component.
Narrative
JSON
ABC License Verification — California state cultivation license issued by the California Department of Food and Agriculture (CDFA).

Verification requires: issue date, expiration date, license number, active/good-standing status, and an evidence source URL. Refreshed monthly. Required for all tiers (Tier 1, 2, 3).

On failure: flag for manual review at severity HIGH (S3).
JSON (machine-readable — same data, evaluator format):
{"id":"license.ca_cultivation_cdfa","type":"license_verification","label":"ABC License Verification", ...}
This is the two-layer policy model. The narrative is what compliance staff read and review. The JSON is what the evaluator runs during a scan. Both are authoritative — editing either view updates the other. The narrative is auto-generated from the JSON on save; manual edits to the narrative are diffed and synced back to the JSON on next save.
