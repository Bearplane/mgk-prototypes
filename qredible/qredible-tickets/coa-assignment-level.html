1 — COA Assignment Level Values + Badge Guide

Individual (default)
This product requires its own COA. No inheritance. Standard behavior for standalone products.
COA Status → COA Validated or Missing solid outline = direct assignment
Child
Each child variant requires its own COA. The parent container row shows a rolled-up status derived from children. Parent cannot have a COA assigned directly.
Parent shows → COA Validated ⤵ or Missing ⤵ dashed = inherited/rolled-up
Inherited
This variant does NOT require its own COA. It inherits COA status from a designated sibling variant (first qualifying Individual variant in the same parent). Does not count toward Missing totals.
Shows → COA Validated ⤵ dashed = inherited from sibling
Bundle
Product is composed of other products in the catalog. Does not require its own COA. COA status derived from component products. Does not count toward Missing totals.
Shows → COA Validated ⤵ dashed = derived from bundle components
Dashed vs solid badge outline: Any COA Status badge that is inherited or rolled-up (not directly assigned) uses a dashed outline border instead of solid. Hovering shows a tooltip: e.g. "Inherited from: Bubba Kush Hemp Flower 3.5g (SKU: 164800-SMO-4)"
