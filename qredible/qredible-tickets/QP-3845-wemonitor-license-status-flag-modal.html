We Monitor — License Status column redesign

License Status now shows one of four states determined by priority. Click any non-blank flag to open the License Detail modal (State B). The old hover-only multi-badge tooltip is gone.

🔄
Four-state flag — priority order
① Violative — any license has a violative State License Status (Expired, Suspended, Revoked, Required Not Held, etc.)
② Pending — no violative licenses, but ≥1 license verification task exists in the To Do column of Q-Audit for this merchant
③ Compliant — no violative or pending, and ≥1 license has been verified non-violative
④ Blank — no license verification task exists at all; cell is empty, click is a no-op

Before: hover tooltip showed per-DBA status badges (confusing with multiple statuses). Clicking navigated to a broken Findings tab.
After: single pill click opens License Detail modal (see State B). Blank cell shows empty-state hint.
⚠ Terminology note

"Pending" exists at two independent levels — they must not be confused:

  • Merchant flag = Pending (amber) — means ≥1 open verification task sits in the Q-Audit To Do column. This is purely a workflow state. It says nothing about what the license values are.
  • State License Status field value = "Pending" — means a license application has been submitted but the state has not yet issued it. Per the classification table this counts as Violative at the roll-up level (no valid license held), and its per-card indicator renders red.

Full classification: Active → Compliant · Not Required → Compliant · Unknown → Compliant (open question) · Inactive / Expired / Suspended / Revoked / Pending / Required Not Held → Violative
