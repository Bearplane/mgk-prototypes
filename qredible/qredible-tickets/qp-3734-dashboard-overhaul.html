Dashboard Overhaul — Premier Feature Widgets + System Priorities

Reworked home dashboard: a system-priority banner across the top, then a grid of Premier feature widgets (MyCOA, Q-Monitor / Q-ScanAI, Compliance Portal), always-on Platform widgets, and an admin override view.

Ticket Scope

QP-3734 reorganizes the dashboard around three widget tiers:

  • System-priority — Verification, Expired COAs, New messages, Onboarding — always visible, top banner.
  • Premier feature widgets — MyCOA, Q-Monitor (incl. Q-ScanAI), Compliance Portal. Render or upsell based on subscription.
  • Platform widgets — Tips, In The News, Verification snapshot — always visible regardless of subscription.

Internal / Admin users see all widgets as if every subscription is active.

Annotations & Responsive Behavior

  • Widget classes — Premier (red badge, subscription-gated), System (amber, always top), Platform (slate, always on), Upsell (ink, deep-link to paywall).
  • Premier gating rule — if user.subscriptions.includes(feature) render widget; else if feature.upsellEnabled render upsell; else hide.
  • Internal / Admin override — role === 'internal' || role === 'admin' bypasses all subscription checks; all Premier widgets render in aggregate (platform-wide) mode.
  • Compliance Portal upsell — CTA deep-links to QP-3733 paywall with ?feature=compliance-portal pre-selected.
  • Responsive stacking — 12-col grid collapses to 6-col at ≤ 1000px, then 1-col at ≤ 700px. System banner collapses 4 → 2 → 2. Premier widgets stack full-width on mobile; Platform widgets remain below them.
  • Greeting — Hello, {firstName}! pulled from user profile; falls back to Hello! if unknown.
