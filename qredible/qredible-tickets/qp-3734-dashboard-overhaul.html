Widget behaviour rules

What QP-3734 defines
  • One widget per Qredible feature — MyCOA, Q-Monitor/Q-ScanAI, Compliance Portal. Order matches Q-Config subscription-feature-selector.
  • Subscribed state — widget shows a live KPI (expiring COAs, flagged listings, etc.) with a ghost CTA that deep-links into the feature's tab.
  • Unsubscribed state — widget shows the feature name, 3-4 bullet features (locked icon), and a primary "Upgrade to X" CTA that launches the MyCOA paywall (QP-3733) or respective upsell flow.
  • Coming-soon state — for features not yet released (e.g., Compliance Portal v2). Ghost "Notify me" CTA.
  • No removals — current expired-COA list, verify-company, Tips, and progress cards all remain. Top-right of the dashboard is not clipped — widget row has its own band below Tips.
