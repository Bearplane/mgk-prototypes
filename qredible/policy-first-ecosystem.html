Policy Document = Single Source of Truth

Compliance team authors policies. Engineering builds the renderer/evaluator once. New industries = new policy doc, not new code. This is the architecture we're building toward.

4 New Database Tables

  • pl_policy Every policy document and its history. The "rulebook library".
  • pl_merchant_policies Which merchants use which policies. A merchant-to-rulebook link.
  • pl_policy_coverage_signature What evidence each policy expects — a fingerprint of required signals.
  • pl_compliance_coverage_audit Append-only log of every AI decision. The SOC 2 audit trail.

3 New Services

  • EvidenceTaggingService Phase 0–1 Labels every piece of scan evidence with which rule it answers.
  • PolicyLookupService Phase 1 Fetches the right compliance rules for each merchant automatically.
  • ComplianceCoverageValidator Phase 2 Confirms every required check has been done before a report goes out.

3 Existing Hooks (wire up)

  • json-rules-engine installed, unused Off-the-shelf rule processor. Already in our codebase — we just aren't using it yet.
  • resolveCoaValidationRule() single-line swap One function decides which lab tests to require. Today it reads tier config; switch it to read the policy.
  • POLICY_VIOLATION enum start emitting Label on findings that says which rule broke. Today every finding is tagged generically; we need to tag with the specific rule.
